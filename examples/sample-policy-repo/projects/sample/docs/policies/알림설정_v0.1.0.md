# 알림설정 v0.1.0

| 버전 | 일자 | 변경 |
|---|---|---|
| v0.1.0 | 2026-05-20 | 초기 |

## §1. 알림 채널

### §1-1. 지원 채널

- 이메일 · 브라우저 푸시 · 슬랙 웹훅 (기업 플랜) · SMS (별도 요금)

### §1-2. 기본 활성

- 이메일 → 기본 ON (계정 이메일)
- 브라우저 푸시 → 기본 OFF (첫 진입 시 권한 요청 dialog)
- 슬랙 · SMS → 기본 OFF (사용자 명시 설정 필요)

## §2. 알림 유형

### §2-1. 필수 알림 (수신 거부 불가)

- 결제 완료 · 실패 · 환불
- 보안: 로그인 위치 이상 · 비밀번호 변경 · 2FA 활성/비활성

### §2-2. 선택 알림 (수신 거부 가능)

- 캠페인 발송 완료 리포트 · 캠페인 발송 실패 alert
- 잔액 임계치 경고 (기본 : 잔액 < 10,000 P 시)
- 신규 기능 안내 · 프로모션

### §2-3. 설정 화면

- 사용자 페이지 우측 톱니바퀴 → 알림 탭
- 좌측 : 유형별 리스트 (그룹 접기 가능)
- 우측 : 채널별 on/off 체크박스 grid
- 저장 시 즉시 반영 (별도 저장 버튼 X)

## §3. 슬랙 웹훅 연동

### §3-1. 설정 방법

1. 슬랙 워크스페이스에서 Incoming Webhook URL 생성
2. 앱 설정 페이지에서 URL 입력 + [테스트 알림 보내기] 버튼 → 슬랙 채널에 테스트 메시지 도착 확인

### §3-2. 실패 처리

- URL 유효성 검증 실패 → 즉시 error message ("유효하지 않은 웹훅 URL")
- 실제 발송 실패 3회 연속 → 슬랙 연동 자동 해제 + 사용자에게 이메일 안내
