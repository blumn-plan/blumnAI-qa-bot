## §1. 잔액·충전

### §1-1. 잔액 표시

- 상단 헤더 우측 : 💰 아이콘 + 현재 포인트 · 소수점 0자리 · `1,234 P` 형식 (천 단위 콤마)
- 잔액 < 발송 예상 금액이면 → 대시보드 §2-1 주황 배너 자동 표시

### §1-2. 충전 단위

- **최소 5,000 P · 최대 500,000 P** 단일 결제
- 프리셋: 5,000 / 10,000 / 30,000 / 50,000 / 100,000
- 커스텀 입력 시 100 P 단위로 자동 반올림 + toast 안내

## §2. 결제 흐름

### §2-1. 결제 수단

- 신용카드 (토스페이먼츠 위임)
- 계좌이체 (실시간)
- 회사 카드 (승인 필요 — 결제 후 관리자 알림)

### §2-2. 결제 실패 처리

- 실패 즉시 modal: "결제에 실패했습니다. 다른 카드로 시도하거나 관리자에게 문의하세요."
- 자동 재시도 X (사용자 명시 재시도만)
- 실패 이력 30일 보관 · 관리자 페이지에서 조회 가능

## §3. 환불

### §3-1. 자동 환불 조건

- 결제 후 24시간 이내 + 사용 이력 0건 → 앱 안에서 [환불 요청] 버튼 활성
- 이외 조건 → 자동 환불 불가. 관리자 문의 안내

### §3-2. 수동 환불

- 관리자 승인 필수 (SLA 영업일 3일)
- 승인 시 결제 수단 원복 (신용카드는 3-5영업일, 계좌이체는 즉시)
- 환불 완료 시 사용자에게 이메일 + 앱 알림

### §3-3. 부분 환불

- 미지원 (전체 환불만)

## §4. 잔액 사용 정책

### §4-1. 발송 단가

- SMS 20 P · LMS 40 P · MMS 90 P · 알림톡 8 P
- 실시간 산정 → 발송 시 잔액 즉시 차감

### §4-2. 잔액 부족 시 발송

- **발송 예약 상태로 대기** (하이라이트 노랑)
- 사용자에게 알림 : "잔액 부족으로 대기 중. 충전 시 자동 발송"
- 24시간 내 미충전 시 자동 취소 → 다시 예약 필요
