## §1. 화면 구성

캠페인 리스트 페이지는 다음 요소로 구성된다:

- 상단 툴바 — [+ 새 캠페인] 파랑 버튼 · 검색 input · 상태 필터 드롭다운
- 리스트 테이블 — 캠페인명 · 상태 뱃지 · 발송일 · 예약 수신자 수 · 액션
- 하단 페이지네이션 — 20건 단위

## §2. 상태 뱃지 정책

### §2-1. 표시 규칙

- **초록 "발송 완료"** — 발송 성공률 95% 이상
- **주황 "부분 실패"** — 성공률 60-95%
- **빨강 "발송 실패"** — 성공률 60% 미만
- **회색 "예약"** — 아직 발송 시각 미도래
- **파랑 "발송 중"** — 발송 진행 중 (스피너 아이콘 + 실시간 진행률)

### §2-2. 뱃지 클릭 동작

- 발송 완료·부분 실패·발송 실패 → 상세 리포트 페이지로 이동
- 예약 → 예약 편집 모달 오픈
- 발송 중 → 실시간 로그 오버레이

## §3. 캠페인 액션

### §3-1. 편집

- 예약 상태만 편집 가능 (발송 중·완료는 편집 잠금)
- 편집 저장 시 서버 검증 (수신자 리스트 최신 여부) 후 반영

### §3-2. 삭제

- **삭제 확인 모달 필수** — "정말 삭제하시겠어요? 이 작업은 복원할 수 없습니다." 메시지 + [취소] / [빨강 삭제] 버튼
- 발송 중 캠페인은 삭제 불가 (모달 대신 toast: "발송 중인 캠페인은 삭제할 수 없습니다")
- 삭제 시 소프트 삭제 (`is_deleted=1`) — 30일 후 hard delete

## §4. 검색·필터

### §4-1. 검색

- 캠페인명 · 태그 · 발신자명 대상
- 최소 2자 입력 시 즉시 검색 (debounce 300ms)
- 검색 결과 0개 시 empty state ("일치하는 캠페인 없음" + [+ 새 캠페인 만들기] CTA)

### §4-2. 필터

- 상태 다중 선택 가능 (기본: 발송 완료 제외)
- 날짜 범위 선택 (기본: 최근 30일)
- 필터 상태는 URL 쿼리로 반영 → 새로고침·공유해도 유지
