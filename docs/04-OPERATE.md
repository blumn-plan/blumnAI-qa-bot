# 04. 운영 (Operate)

협업자(QA·운영)·기획자가 매일 봇을 어떻게 쓰는지. 셋업이 끝난 뒤 일상 사용법.

---

## 두 진입점

| 누가 | URL | 비번 |
|---|---|---|
| **협업자** (QA·개발자·운영자) | `https://<pages_url>/.blumnAI-qa-bot/apps/qa-collab.html` | 없음 |
| **기획자** (정책 변경 권한자) | `https://<pages_url>/.blumnAI-qa-bot/apps/qa-planner.html` | 있음 (코어 메인테이너에게 받기) |

URL 은 본인 팀 `blumnAI-qa-bot.config.yml` 의 `deployment.pages_url` 기준.

---

## 협업자 흐름 — qa-collab.html

### 1) 첫 진입 — 이름 설정

처음 들어가면 **👤 사용자 설정** 다이얼로그 → 이름(또는 닉네임) 입력 → 저장.

본인 PC 의 localStorage 에만 저장. 전달이력·개선건 작성자 식별용으로만 사용. 우상단 [변경] 으로 언제든 수정.

### 2) 프로젝트 + 문서 고르기

좌측 사이드바:
- **프로젝트** select — `blumnAI-qa-bot.config.yml` 의 `projects[]` 목록
- **보기 범위** — `👤 내것만` (기본) / `🌐 전체` — 전달이력·개선건 필터

좌측 본문(섹션별 접기):
- **정책문서** — 정책 md 목록
- **화면설계서** — storyboard md 목록 (있는 경우)
- **📜 최근 대화** — 이 PC 에서 했던 대화 (제목·본문 검색 가능)
- **📤 기획자 전달이력** — 변경 요청 목록 (📋 대기 / ✅ 적용 / 🚫 보류 + 기간 필터)
- **📝 AI봇 답변 개선건** — 누적된 개선 룰 (일자 + 질문자 + 요청 내용)

문서 1개 클릭 → 가운데 영역에 렌더링.

### 3) AI 에 질문

우측 채팅 영역에 질문 입력 → Enter (Shift+Enter 로 줄바꿈).

답변은 **선택한 문서를 근거로** 옴. 답변 끝의 `§X-Y` 는 클릭 가능 — 가운데 영역의 해당 절로 자동 스크롤 + 노란 하이라이트.

### 4) 답변 받은 뒤 — 3가지 분기

#### 분기 A. 답변이 정확하고 추가 변경 불필요

- [↻ 새 대화] 클릭 후 다음 질문으로 / 또는 그대로 탭 닫기
- 별도 OK 작성 불필요 — 채팅 자체가 종결 신호

#### 분기 B. 답변 스타일·표현이 불편 (정책 변경 X, AI 답변 품질 개선)

답변 카드의 [📝 **개선 요청**] 버튼 클릭 → 다이얼로그:

| 입력 칸 | 설명 |
|---|---|
| 개선 요청 사항 (필수) | 예: "5줄 안으로 요약", "기술용어 빼고", "구체 예시 1개" |
| 짧은 제목 (선택) | 파일명에 사용 |
| ☑ 저장 후 이 답변 다시 받기 | 기본 ON. [저장] 누르면 즉시 개선 반영 재답변 (🔁 태그 + 초록 테두리) |

저장되면:
- `qa/feedback/YYYY-MM-DD-...md` 에 기록 (main 직접 commit)
- 답변 카드 버튼이 `✓ 개선 요청 전달됨` 으로 변경
- 이후 **모든 답변** 작성 시 누적 개선 룰을 우선 참고 → 같은 실수 반복 안 함

#### 분기 C. 정책 자체를 바꿔야 한다 (모호함·정합성·신규 정의)

채팅 하단 [📤 **기획자에게 전달**] (초록색) 클릭 → 다이얼로그:

| 입력 칸 | 설명 |
|---|---|
| 변경 요청 제목 (필수) | 전달이력 목록에 노출. 예: "대시보드 필터 초기화 버튼 위치 변경" |
| 어디에 어떻게 반영해 달라 (필수) | 위치·변경·사유 명시. 예: "대시보드_v0.1.5.md §3-2 [필터 영역] — 현재 활성 시에만 노출 → 항상 노출로 변경. 사유: 사용자 혼동 줄임" |

[생성하기] 누르면:
- `qa/decisions/YYYY-MM-DD-짧은제목.md` 에 기록 + 채팅 전문도 함께 저장
- 좌측 사이드바 **📤 기획자 전달이력** 에 📋 대기 상태로 추가
- 기획자가 qa-planner.html 에서 처리할 때까지 대기

### 5) 기획자가 본인 요청 처리하면 알림

다음 qa.html 접속 시 우측 채팅 상단 초록 배너:
- 예: `기획자가 "[대시보드 필터 초기화 버튼 위치 변경]" 을(를) 적용 처리했습니다`
- [자세히] → 해당 전달이력 카드로 점프 (처리 사유·반영 위치 확인)
- [확인] → 배너 닫힘

## 기획자 흐름 — qa-planner.html

협업자가 [📤 기획자에게 전달] 누른 변경 요청을 모아 검토·처리.

### 접근

URL 입력 → 비밀번호 입력. (비번 잊으면 코어 메인테이너에게 문의)

### 화면 구성

| 영역 | 내용 |
|---|---|
| 좌측 리스트 | 전달이력 (상태 필터 + 검색) |
| 가운데 본문 | 선택한 합의문 markdown 렌더 |
| 하단 액션 | ✅ 적용함으로 표시 / 🚫 보류로 표시 / ↺ 대기로 되돌리기 / 📋 본문 복사 / 📄 관련 정책 열기 / §B-2 프롬프트 복사 |

### 처리 절차

#### A. 단순 적용 (정책 본문 직접 수정 안 함)

1. 좌측에서 합의문 선택
2. 본문 검토 → [✅ **적용함으로 표시**] 클릭
3. 메모 다이얼로그:
   - **그대로 적용** → 상태만 `✅ {date} 적용` 으로
   - **메모 저장하고 적용** → 메모를 합의문 본문에 박음 + 상태 `✅ {date} 적용 (메모 있음)` — **협업자가 다음 진입 시 메모 확인 가능**

#### B. 정책 본문에 반영하기

1. 합의문 본문 검토
2. [📄 **관련 정책 열기**] → 새 탭에서 정책 md 렌더
3. [📋 **본문만 복사**] → 합의문의 "✅ 변경 후" 텍스트 클립보드
4. 정책 md 에 직접 patch bump 적용 ([02-WIRE-POLICIES.md §정책 변경 절차](02-WIRE-POLICIES.md#정책-변경-절차--2가지-규칙) 참고)
5. 반영 끝나면 [✅ 적용함으로 표시]

#### C. Claude Code 에 위임해서 자동 반영

1. [§B-2 프롬프트 복사] 클릭 → 클립보드에 다음 프롬프트 복사됨:
   ```
   위 합의문(qa/decisions/) 의 "✅ 변경 후" 본문을 §B-2 수행 순서대로 적용:
   ...
   ```
2. VS Code 의 Claude Code (Antigravity) 에 붙여넣기 → 자동으로 patch bump + 박제 + cross-link 갱신 + 코드 수정까지
3. 결과 확인 → [✅ 적용함으로 표시] 또는 사용자 명시 시 배포

#### D. 보류

1. [🚫 **보류로 표시**] 클릭
2. 사유 입력 (예: "v0.2.0 출시 후 재검토")
3. 상태 `🚫 보류 ({사유})` 로 저장 — 협업자에게도 노출

#### E. 되돌리기

`↺ 대기로 되돌리기` — 잘못 처리한 경우 다시 📋 대기로 복귀.

### 알림 받기

- 우상단 [🔔 알림 받기] 클릭 → 브라우저 알림 권한 허용
- 새 변경 요청이 들어오면 OS 알림으로 즉시 인지
- qa-planner.html 탭 닫혀 있어도 동작 (브라우저는 켜져 있어야)

## 자주 묻는 질문

### Q. 답변이 이상해요

**A**. 3가지 원인:
1. **정책 본문 부실** — 그 질문의 답이 정책에 없거나 모호함. → 협업자가 [📤 기획자에게 전달] 로 정책 보완 요청
2. **답변 스타일 문제** — 너무 길거나 기술용어 많음 → 협업자가 [📝 개선 요청] 으로 룰 추가
3. **C 모드 PC 끊김** — 답변 자체가 안 오거나 "PC 서버 꺼져있음" 에러 → 호스트에게 `start.bat` 재실행 요청

### Q. 변경 요청 보냈는데 기획자가 아무 응답 없어요

**A**. qa-planner.html 의 알림 권한이 꺼져 있을 가능성. 직접 Slack/메신저로 한 번 더 알려주세요. 본인 합의문 URL 을 공유하면 즉시 점프 가능.

### Q. 답변 캐시되나요?

**A**. 안 됨. 같은 질문 두 번 하면 두 번 다 API 호출. C 모드면 본인 PC 의 Claude CLI 가 5시간 한도 내에서 무료 처리.

### Q. 우리 팀 정책이 100개 넘는데 봇이 다 알고 답하나요?

**A**. 봇은 **선택한 문서 1개 + 답변 규칙(answer-rules.md) + 최근 피드백** 만 컨텍스트로 받음. 다른 정책은 모름. 정책 간 cross-link 가 필요한 질문은 협업자가 두 정책을 모두 검토 후 종합 판단.

### Q. 이전 대화 기록 누구한테 보이나요?

**A**. 본인 PC 의 localStorage 에만 저장. 다른 사람은 못 봄. 단 [📤 기획자에게 전달] 누른 합의문은 GitHub 에 commit 되므로 레포 접근권 있는 모든 팀원이 봄.

### Q. 비용은?

**A**. C 모드면 0원. A 모드면 월 100질문 기준 약 $15-30 (`claude-sonnet-4-6` 기준). Anthropic Console 의 Usage 페이지에서 실시간 확인.
