# 00. 개요 (Overview)

**blumnAI-qa-bot** 은 회사 안의 다른 팀이 본인 정책문서를 가지고 AI 봇과 대화하며, 변경 요청을 자동 합의문화할 수 있게 해주는 시스템입니다.

## 무엇을 해주나

QA·운영팀이 매일 겪는 문제 — *"이 화면이 정책상 어떻게 동작해야 하는지 누구한테 물어야 하지?"* — 를 해결합니다.

1. **정책 질문 → AI 가 즉시 답변**
   협업자(QA·운영)는 정책문서·화면설계서를 보며 채팅창에 질문. 봇이 그 문서를 근거로 답해줌. (예: *"대시보드 알림 배너 닫기 동작 어떻게 돼?"* → *"주황 배너는 닫으면 24시간 미표시. 빨강 배너는 닫기 불가. §2-2 참고"*)

2. **답변이 애매하거나 정책 자체를 바꿔야 하면 → [📤 기획자에게 전달]**
   기획자가 검토 후 ✅ 적용 / 🚫 보류 처리. 모든 합의는 `qa/decisions/` 에 markdown 으로 자동 저장.

3. **답변 톤이 불편하면 → [📝 개선 요청]**
   *"5줄 안으로 요약해줘", "기술용어 빼고"* 같은 요청. `qa/feedback/` 에 누적되어 이후 답변에 자동 반영.

## 아키텍처 한 장

```
┌─────────────────────────────────────────────────────────────┐
│ 협업자 PC                                                    │
│ ┌────────────────┐                                          │
│ │ 브라우저       │  qa-collab.html  (GitHub Pages 호스팅)     │
│ │                │  qa-planner.html (기획자만)               │
│ └───────┬────────┘                                          │
└─────────┼───────────────────────────────────────────────────┘
          │
          ▼  (fetch)
┌─────────────────────────────────────────────────────────────┐
│ Cloudflare Worker  (팀당 1개, 무료 tier)                     │
│   GET  /list-projects · /list-docs · /doc                   │
│   POST /qa · /forward · /feedback                           │
└─────────┬───────────────────────────────────────────────────┘
          │
          ├─ A 모드 ─→  Anthropic API (sk-ant-... 유료, 월 ~$30)
          │
          └─ C 모드 ─→  본인 PC tunnel
                          └─ Claude Code CLI (Max 구독, 무료)
                          └─ GitHub API (정책 read, decisions write)
```

## 두 가지 운영 모드 — C 모드 추천

| | A 모드 (유료) | **C 모드 (추천)** |
|---|---|---|
| 비용 | API 토큰당 과금. 월 100질문 ≈ $15-30 | **0원** (본인 Max 구독 한도 안에서) |
| PC 의존성 | 없음 (Worker 가 다 처리) | **PC + `start.bat` 콘솔창** 항상 켜져 있어야 |
| 응답 속도 | 3-5초 | 8-15초 (claude CLI 오버헤드) |
| 셋업 난이도 | 회사 Anthropic 결제 받기 어려움 | Max 구독만 있으면 됨 (개인 또는 회사) |
| 미래 위험 | 없음 | Anthropic 정책 변경 시 막힐 수 있음 |

→ **C 모드로 시작, 가치 입증되면 A 모드 전환** 이 일반 추천 경로.

## 사용자 레포 = 정책 SSoT

봇 코드는 본인 팀 레포에 깔리는 게 아니라, **본인 팀이 만든 GitHub 정책 레포** 에 컨피그 파일 하나로 연결됩니다. 즉:

- 우리 코어 레포(`blumnAI-qa-bot`) — Worker 코드, HTML, 가이드
- 광고팀 정책 레포(`ad-team-policies` — 광고팀이 만듦) — `blumnAI-qa-bot.config.yml` + 정책 md + qa/decisions/

광고팀은 컨피그 1개 채우고 코어 코드 사본을 본인 레포에 깔면 끝.

## 무엇을 얻나

- **즉답** — 정책 질문 → 챗박스 한 번 → 7초 답변. Slack 에서 누구 찾아갈 필요 없음
- **자동 합의 기록** — 변경 요청을 입으로만 약속하지 않음. `qa/decisions/` 에 markdown 으로 박힘. 누가 언제 무엇을 왜 바꿔달라고 했고, 기획자가 어떻게 처리했는지 git history 로 추적 가능
- **답변 품질 자가 학습** — 한 번 받은 개선 요청은 이후 모든 답변에 자동 반영. 같은 실수 반복 안 함
