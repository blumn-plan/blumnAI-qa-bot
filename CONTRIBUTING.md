# 코어 개선 기여 가이드 (Contributing)

이 문서는 **코어 봇 (`blumn-plan/blumnAI-qa-bot`) 에 개선을 제안하고 싶은 팀 담당자용** 입니다. 각 팀이 자기 정책 레포 안에서 봇을 운영하는 방법은 [README](README.md) 와 [docs/](docs/) 참고.

---

## 핵심 원칙 — 자동 반영 X, PR + 리뷰만

- ❌ **팀 사본 (`.blumnAI-qa-bot/`) 의 수정은 원본에 자동으로 흘러가지 않습니다** — 팀이 자유롭게 수정해도 원본은 안전
- ✅ **코어에 반영하려면 반드시 PR + 기획팀 리뷰** — 검증되지 않은 변경이 모든 팀에 배포되는 사고 방지
- 🔒 **main 브랜치 보호 활성** — 직접 push 불가. `.github/CODEOWNERS` 에 따라 파일 유형별 리뷰 요구

---

## 언제 코어에 기여하나

**팀 사본만 고쳐도 되는 경우** (PR 필요 X):
- 팀 정책 md 수정 → 팀 레포에서 처리
- 팀 답변 규칙 (`.blumnAI-qa-bot/answer-rules.md`) 수정 → 팀 레포에서 처리
- `blumnAI-qa-bot.config.yml` 값 조정 → 팀 레포에서 처리
- 팀 자체 workflow · 개선 요청 → 팀 레포에서 처리

**코어에 PR 이 필요한 경우**:
- 봇 실행 코드 개선 (`apps/`, `bot/worker/`, `bot/local-server/`)
- 새 기능 추가 (예: 새 UI 요소, 새 엔드포인트, 새 알림 방식)
- 답변 규칙 템플릿 개선 (`examples/sample-policy-repo/.blumnAI-qa-bot/answer-rules.md`)
- 가이드 문서 개선 (`docs/`, `README.md`)
- CI · 릴리즈 워크플로우 개선 (`.github/workflows/`)

## PR 절차

### 1. Issue 로 먼저 문제·아이디어 공유 (권장)

`blumn-plan/blumnAI-qa-bot` 레포에 Issue 오픈:
- **어떤 문제**: 팀 사용 중 어떤 상황에서 무엇이 부족했는지
- **제안 해결**: 어떻게 고치면 좋을지 (초안·아이디어)
- **영향 범위**: 다른 팀에도 영향 있는지, 팀 별 옵트인이 필요한지

기획팀이 방향을 승인하면 PR 로 진행.

### 2. Fork + 브랜치

- 코어 레포를 본인 계정으로 fork
- 새 브랜치: `feat/<간단설명>`, `fix/<간단설명>`, `docs/<간단설명>`
- ⚠️ 팀 정책 md · 시크릿 · 개인 정보는 절대 커밋 X (Issue 스크린샷도 개인 정보 없는지 확인)

### 3. 개발·자체 검증

- 코드 변경 시 `bot/worker/` 는 `npx tsc --noEmit` 통과
- HTML 변경 시 `apps/*.html` 을 로컬에서 열어서 동작 확인 (또는 F12 콘솔 에러 없음)
- 문서 변경 시 GitHub 프리뷰로 마크다운 렌더 확인
- **팀 사본에서 dogfooding 완료 후** PR 열기 — 최소 24시간 실사용 검증 권장

### 4. PR 열기

원본 레포 `main` 대상으로 PR 오픈. 본문에 다음 5가지 명시:

```markdown
## 무엇을 바꿨나
(1-3줄 요약)

## 왜 필요한가
(어떤 팀·시나리오에서 문제가 있었는지)

## 어떻게 검증했나
- [ ] 팀 사본에서 dogfooding (며칠, 몇 명이 사용했는지)
- [ ] 자체 테스트 (스크린샷·로그 첨부)
- [ ] 다른 팀 영향 검토 (breaking 있으면 CHANGELOG ⚠️ 표기)

## 관련 Issue
Fixes #NN 또는 참조 링크

## Reviewer 메모
(리뷰어가 특별히 봐줬으면 하는 부분)
```

### 5. 리뷰 대응

- CODEOWNERS 기반으로 GitHub 이 리뷰어 자동 요청 (기획팀·개발팀)
- 리뷰 코멘트에 응답. 필요 시 커밋 추가 (force-push 지양)
- 승인 받으면 기획팀이 머지 (본인이 머지 X)

### 6. 배포·팀 전파

머지 후:
- 기획팀이 새 버전 태그 (예: `v0.1.1`) + CHANGELOG 정리 + GitHub 릴리즈 publish
- 자동 알림 (배너 · Teams 웹훅) 이 각 팀에게 발송
- 각 팀은 🟠 업데이트하기 프롬프트로 사본 갱신

## SLA

- **첫 응답**: 영업일 3일 내
- **작은 변경 (문서·오타)**: 1주일 내 머지 가능성 높음
- **큰 변경 (코드·기능)**: 리뷰·조율 필요해서 2-4주 소요 가능
- **긴급 (SECURITY)**: 24시간 내 첫 응답 · 48시간 내 머지 목표

## 도움 요청

- 코어 아키텍처 궁금하면 → [docs/00-OVERVIEW.md](docs/00-OVERVIEW.md)
- PR 여는 법 모르겠으면 → 기획팀 Slack 채널에 질문 or 위 Issue 로 상담 요청
- 팀 담당자가 개발자 아니어도 OK — Issue 로 아이디어만 남겨주시면 기획팀이 PR 대신 열어드립니다
